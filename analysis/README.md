# analysis/

Working notes from individual benchmark runs — results triage, flagged-fixture diagnosis,
go/no-go calls on whether to scale a pilot. These keep the repo root reserved for the
front-door documents (`README.md`, `PAPER.md`, `PREREGISTRATION.md`, `ABC_CHECKLIST.md`).

Each note is tied to a specific run under `results/` and names it at the top. Notes are
descriptive, not authoritative: the paper and pre-registration are the sources of record.

- [`HAIKU_MULTI_PILOT_ANALYSIS.md`](HAIKU_MULTI_PILOT_ANALYSIS.md) — multi-turn haiku pilot
  (`results/20260615T200423Z/`): verdict to pursue the full matrix, plus triage of the four
  oracle-flagged scenarios.
